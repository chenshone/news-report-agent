# 概述
热点资讯聚合 Agentic AI 是一个能够自主获取每日热点资讯、过滤甄别信息并深入解读分析的智能代理系统。不同于传统一次性生成回答的方式，Agentic AI 采用多步循环的“代理式工作流程”（Agentic Workflow），使 AI 能像人一样逐步思考、计划、检索、反思并改进结果。
本方案的核心目标是打造一个能够模拟人类专家认知流程的数字智能体：它不仅能根据用户意图制定检索策略，还能在执行过程中通过“自我反思”剔除虚假或低质信息，并在发现信息缺口时自动发起补充检索（Recursive Planning），最终通过多专家协作模式输出具有深度的解读报告。
该智能体根据用户的输入需求，从多个平台和网络来源抓取最新的热点内容。系统会剔除虚假、不相关或低质的信息，只保留有价值的新闻和话题进行深入分析解读，最终输出包含详细内容和多维度解读的报告（附参考链接），帮助用户快速了解热点背后的深层意义。整个流程将严格遵循 Agentic AI 理念：在接受任务后先规划步骤，利用工具检索信息，借助多代理协作多角度分析，并通过反思机制不断优化最终输出。

# 需求与目标
- 领域可定制：用户可以动态指定感兴趣的资讯领域或主题作为智能体的输入（例如科技、财经、娱乐等），系统据此调整检索关键词和关注方向，确保获取的内容符合用户期望。该功能在最终产品中是必要的（允许对话式地输入关注点），但在 MVP 阶段可先固定一个领域进行验证。
- 定时自动运行：智能体应能够每日定时触发（例如每天早上9点）运行一次任务，自动抓取过去24小时内各平台的热点资讯。可以通过本地调度（如 Cron 作业或调度脚本）来定时调用代理执行，使其成为一个持续更新的新闻助手。
- 多源信息检索：系统需要集成多个信息源获取热点内容，包括通用网络搜索引擎（如Google）以及特定平台（如小红书、抖音等）的搜索或热点接口。通过多渠道获取，确保资讯的全面性和多样性。每个来源可视为工具或子代理，提供不同类型的信息（新闻报道、社交媒体趋势、视频热度等）。
- 快速筛选与真实性检查：在检索到初步内容后，代理应能依据标题和摘要对结果进行快速评估，剔除不相关、重复或低质量（虚假夸大、不可靠来源）的信息。筛选标准包括与指定领域的相关度、内容可信度（例如是否来自权威来源或多方报道验证）以及话题热度。只有通过筛选的条目才进入深入解读阶段。这一步骤也包含代理的反思机制——如果发现通过筛选的有价值结果太少，代理会反思可能的原因，如检索关键词不当或近期确实缺乏重大新闻，并相应调整查询策略（扩展/修改关键词或切换其它来源）以获取更多有价值信息。
- 深度解读与多维分析：对于筛选后的每条资讯，智能体将获取其完整内容（例如新闻全文、帖文全文等）并进行多维度深入分析解读。这里将引入多智能体协作模式：系统为不同分析维度配备专门的子代理，就如同一个小型专家团队对同一内容进行会商。例如，对一篇新闻，可能由若干子代理分别从以下角度进行解读：内容摘要与要点提炼、事实核查与背景补充、影响力和舆论分析、相关后续发展预测等。每个子代理在各自隔离的上下文中独立工作，发挥其专长，对资讯进行深入剖析和评价。这种多代理协作相当于不同智能体角色相互检查与补充，从而进一步提升结果质量。子代理分析过程中如果遇到信息空白或需要外部数据支撑的地方，可以再次调用工具（例如再次搜索相关背景资料），形成检索-分析的闭环，确保解读的全面性。
- 结果讨论与综合：在各子代理完成其维度的分析后，主代理需要整合所有子代理的结论和见解。主代理扮演“编辑”和“审核”的角色，将多方面的信息汇总成连贯的解读报告。在整合过程中，代理应进行反思审查：检查不同来源的信息是否存在矛盾，子代理结论是否有遗漏或需要修正，并调用反思模式对报告草稿进行质量评估。如果发现问题，主代理可以协调子代理进一步讨论或修改，确保最终报告的准确和完整。
- **报告生成与输出格式：**最终输出的报告针对每日精选的几条资讯生成分条目详尽解读。报告将采用富格式 Markdown（并可渲染为网页 HTML）呈现，包括清晰的层次结构和版式，方便阅读。具体要求：
  - 每条入选资讯作为一个小节，包含标题（或简要说明）、内容概述以及深度解读。解读部分综合各子代理观点，形成对该资讯意义的分析评论。
  - 文中嵌入引用链接作为参考来源，以证明内容的真实性和提供延伸阅读（例如引用新闻原始出处或权威解读来源）。引用格式遵循 【来源†行号】 形式，确保信息可追溯。
  - 采用简洁明晰的 Markdown 排版：包括适当的标题层级、段落、项目列表等，使报告易于浏览。段落控制在3-5句话以内，重要信息可以用列表列出。
  - 如果有合适的示意图片或图表且获取方便（例如事件相关图片、数据图表），可嵌入图片增强说明，但 MVP 阶段可不强制要求插图。
- **对话式交互与使用体验：**系统将以对话机器人形式提供入口，用户通过对话输入需求（例如“请给我今天科技行业的热点新闻解读”），智能体接收指令并开始上述流程。整个过程对用户是透明的，一问一答式给出最终报告。在MVP实现中，可在本地以命令行或简单界面运行；后续可集成到对话应用或网页前端，实现更友好的交互。

# 整体架构
整体架构采用模块化的多阶段流水线，由主代理协调各步骤的执行，并在关键节点引入子代理协作。架构设计遵循规划-执行-反馈循环，确保智能体在复杂任务中有条不紊地推进。主要流程包括：任务规划 → 信息检索 → 内容筛选 → 深入解读 → 结果整合 → 报告生成。下面对各阶段进行详细说明：
## 1. 规划阶段
在资讯获取场景中，用户的需求往往是高度概括的（例如：“关注近期 AI 领域的 Agent 进展”）。规划要求 Agent 不直接执行搜索，而是首先充当“战略家”。在收到用户任务后，主代理首先进入规划模式。它需要将宏观任务拆解为具体的子任务清单。
- 查询分解与重组：根据用户的补充需求，Agent 需要识别查询中的核心实体，并生成Query content(一组)。例如，用户输入“今天科技圈发生了什么大事”，则可能制定查询如「今天 科技 新闻 热点」以及更具体的子主题（AI、新产品发布、大公司动态等）以覆盖广度。关键词不局限于一个——代理可能准备多个相关查询，以便搜索不同维度的信息。如果用户输入“特斯拉 Robotaxi 和 Waymo 的安全数据对比”，Planner Agent 必须具备逻辑判断能力，将其拆解为并行的子任务——“检索特斯拉 Robotaxi 最新发布会”与“检索 Waymo 2024年安全事故报告”，或者在发现两者关联紧密时合并查询。
- 动态调整：规划不是一次性的。如果初步检索发现信息不足，Agent 必须有能力修改后续计划。
- 确定信息源与工具使用计划：代理列出需要调用的工具和信息源。例如：使用Google搜索获取全网新闻摘要；调用小红书爬虫/API获取社交平台热点讨论；使用抖音开放接口获取视频热点标题等。每个来源的查询可能略有不同，代理需要为各渠道制定相应检索计划。
- 任务清单生成：代理生成一个有序的TODO列表，列明将要执行的步骤。例如：
  1. 用Google搜索查询1获取新闻列表。
  2. 用Google搜索查询2获取新闻列表。
  3. 用小红书API搜索话题。
  4. 汇总检索结果并筛选。
  5. 针对每条入选新闻启动深度分析子代理。
  6. 汇总所有分析并撰写报告草稿。
  7. 校对并输出最终报告。
  这样的任务清单让代理在复杂流程中保持条理清晰，即使后续过程出现新信息需要动态调整，仍有一个基准计划可循。规划清单本身也会被记录在代理的虚拟文件系统中，便于后续步骤参考和更新。
- 策略自省 (Pre-Search Reflection)：智能体在规划完毕后，对计划进行一次检查。
  - 反思查询的完备性：用户输入可能很模糊。Agent 需要反思：“我生成的这个搜索词（Query）能搜到东西吗？是否太宽泛了？”
  - 多视角规划：对于一个热点，Agent 应反思需要哪些视角。例如，针对“新款手机发布”，它不应只生成一个“参数配置”的搜索词，而应反思出“我还需要搜索‘用户真实评价’和‘竞品对比’才能写出深度报告”。
## 2. 信息检索与工具使用
规划确定后，代理按顺序执行信息检索任务。在DeepAgents框架下，代理可以灵活调用外部工具，实现工具使用模式，即让LLM调用功能完成自身无法直接完成的操作。
- 多模态检索：工具不仅是 API 的调用，更是对不同平台数据形态的适配。对于 Google，工具返回的是标题和摘要；对于小红书，工具需要解析“笔记”结构、提取标签和评论热度；对于抖音，则涉及视频内容的文本化（ASR 或摘要提取）。
- DeepAgents 的文件系统工具：为了处理海量检索内容，系统将引入虚拟文件系统（Virtual Filesystem），允许 Agent 像操作电脑一样 write_file（保存长文）和 read_file（读取特定段落），从而突破 LLM 上下文窗口的限制 。   
检索这一阶段的结果：将得到多来源的候选资讯列表，每条包括基本信息（标题、摘要、链接、发布时间等）。这些数据进入下一步筛选阶段。
## 3. 内容筛选与评估
在获取大量候选资讯后，智能体进入筛选评估环节，以确定哪些内容值得深入分析。此阶段相当于信息质量和相关性的把关人，防止系统浪费精力在无关或不良信息上。这是实现“剔除不相关或不合格内容”的关键。传统的 RAG 系统往往受到“垃圾进，垃圾出（Garbage In, Garbage Out）”的困扰。主要工作包括：
- 多维度质量评估：内容匹配度；内容可信度（标题党？来源媒体，是否存在多条相似信息）；热度等。
- 筛选产出：经过初步过滤后，代理生成“候选深入分析清单”，其中包含通过筛选的若干条新闻/话题及其链接。为了后续处理方便，代理可能为每条入选内容再获取完整正文或主要内容。MVP阶段筛选后只需保留链接和简要信息，正文获取可以稍后进行。
- 反思与调整：
  - 相关性反思（Relevance Check）：检索回来的 10 条结果，可能只有 2 条真正讲的是你关注的事，其他 8 条是广告或 SEO 垃圾站。Agent 需要逐条打分，剔除低分内容。
  - 可信度反思（Credibility Check）：Agent 需要检查来源：“这个域名是权威媒体还是已知的内容农场？”、“标题是否使用了‘震惊’、‘吓尿’等诱导性词汇？” 。   
  - 事实冲突反思：如果两篇文章说法不一（一篇说涨了，一篇说跌了），Agent 需要识别这种冲突，并标记为“需要进一步核实”，而不是盲目采信其中一条。
## 4. 深度解读与多智能体协作
深度解读不能依赖单一视角的模型。对于筛选后的每条资讯，系统启动深度解读流程，旨在对内容进行多角度、多层次的剖析。充分运用多智能体协作与反思两个设计模式，以模拟人类专家团队的讨论分析过程。
- 角色分工：针对一条关于“央行降息”的新闻，系统将根据配置动态生成三个子 Agent：“宏观经济学家”（分析政策背景）、“股市分析师”（预测市场反应）和“普通消费者顾问”（分析对房贷的影响）。
- 子代理的工作流程：每个子代理收到主代理提供的资料（如新闻全文文件路径、相关提示）后，会执行自己的任务链。例如摘要代理只是读取全文然后调用LLM生成摘要；背景代理如果需要额外信息，可以调用搜索工具并将新资料写入临时文件再总结；事实核查代理可能会多次调用搜索核实不同细节；舆情代理可能使用不同工具检索社交媒体内容。子代理完成后，会将自己的分析结果（文字说明、引用等）写入文件或者直接返回给主代理。为了保证子代理输出质量，反思机制也可在子代理层面应用：如事实核查代理对比多方证据后自行确认结论是否可靠，再提交结果；或多个子代理之间可以相互参考检查——例如真实性代理和摘要代理都提到了某数字，但数值不一致时，代理应纠正这一矛盾。
- 多代理讨论与反馈：在所有子代理各自完成后，主代理进入整合前可以执行一次讨论式的反馈过程。这类似于人类专家开会讨论。实现上，主代理可以将所有子代理输出汇总到一个“大文件”或上下文缓冲中，然后让各子代理或一个评审代理再审阅一遍整体内容，检查是否有明显遗漏、冲突或者可以深化的点。例如，舆情代理可能补充指出摘要里遗漏了某个最新进展，背景代理可能纠正事实核查代理对某专业细节的误解等等。这种交叉审阅可通过让LLM阅读汇总并回答检查清单来实现，也可以由主代理逐项对比子代理输出完成。同样，如果发现问题，主代理可以针对某子代理的输出要求它修正并再次提交。经过这番多智能体的协作与反思，最终每条资讯的分析就趋于完善和可靠。
## 5. 结果整合与报告生成
经过深度分析阶段，主代理手中掌握了每条新闻由多个子代理贡献的丰富解读信息。接下来进入最后阶段：将这些信息整合成面向用户的报告。此阶段涉及内容组织、语言生成和最终格式化输出几个部分：
- 逐条整合撰写：主代理以每条资讯为单位进行整合。对于每一条入选新闻，先列出其标题或简单描述，然后将摘要、背景、核查、影响等各方面内容组织成连贯的说明文字。整合时注重逻辑顺序和衔接：通常先简述事件经过（来自摘要代理），接着补充背景知识（背景代理信息），然后说明事件的重要性和影响（影响分析），并穿插任何需要提醒的事实真伪考证（真实性核查结果）。最后若有展望建议也可以附上。主代理应使用清晰的分段和必要的小标题，将长内容拆分为易读的单元。同时保持客观、中立的语气，必要时标明不同观点来源。
- 引用及出处标注：为了增强报告的可信度和参考价值，主代理会将子代理提供的参考资料链接以脚注形式嵌入报告。当某段内容借鉴或引述了具体来源的信息，就在句末添加对应来源引用。这种做法保证最终输出报告中每一重要细节都有据可依，方便用户查证。
- 多条内容的整体结构：如果当天入选了多条新闻，则报告会针对每条新闻分别撰写解析，通常按重要性或时间顺序排列。在报告开头可以有一个引言概述，说明今天关注的主题列表。各新闻板块之间可以加入分隔标题或水平线以区分。也可以在报告结尾附上总体评论或总结，例如“以上是今天 X 领域的主要热点及解读”。
- 语言润色与一致性：在组合多来源内容后，主代理进行最后的语言润色，确保报告读起来流畅、一致，没有重复或前后矛盾。
